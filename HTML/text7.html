Road to ESP32

    • A scripting engine that worked with Tasmota was needed:
      • General purpose programs
      • Ability to write device drivers without recompiling
    • Review of options
      • Micropython - took over the machine
      • LUA - heavyweight and obscure syntax
      • https://github.com/cesanta/elk – Javascript Syntax
      • Berry – Python Syntax

Notes:

Write conventional programs with normal language construction

Device drivers need to interface closely with Tasmota which is providing the usual infrastructure

Micropython would replace Tasmota

LUA consumed too much resources

In the end, the choice came down to the developer preferred pythonese to javascript
